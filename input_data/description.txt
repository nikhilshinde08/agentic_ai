Database Tables and its schemas.

Table descriptions : Patients ( Table Name )
This table holds the list of patients’ details with personal information like SSN, ID, name, birthdate etc.

Columns by description
ID: This column refers to the ID of the patients with the unique value of each record to pull out distinct with similar other tables.
BIRTHDATE: This column refers to the patient’s birth date.
DEATHDATE: This column refers to the patient's death date.
SSN: This column refers to the unique SSN (Social Security Number) of each person.
DRIVERS: This column refers to the unique number of the driving license details of each person.
PASSPORT: This column refers to the unique number of the passport number of each person.
PREFIX: This column refers to pointing out the person with Mr. / Mrs./ Miss.
FIRST: This column refers to the person’s First name.
LAST: This column refers to the person's Last Name.
SUFFIX: This column refers to pointing out the degree or any reputation in their name’s end.
MAIDEN: This column refers to the maiden name of the person who have the Mrs. column filled in the PREFIX column.
MARTIAL: This column refers to the marital status of the person with M and S.
RACE: This column refers to the patient’s identification.
EITHINICITY: This column refers to the patient’s ethnicity.
GENDER: This column refers to the patient’s sex with M or F as Male or Female.
BIRTHPLACE: This column refers to the birthplace of the patient.
ADDRESS: This column refers to the current address location of the patient.
CITY: This column refers to the current city of the patient.
STATE: This column refers to the current state of the patient.
COUNTY: This column refers to the current county location of the patient.
ZIP: This column refers to the zip code of the patient residence location.
LAT: This column refers to the latitude location of the patient location.
LON: This column refers to the longitude of the patient’s location.
HEALTHCARE_EXPENSES: This column refers to the total expenses done from the patient for the clinical.
HEALTHCARE_COVERAGES: This column refers to the total coverages took by the patient from the insurance payer.

Identification fields
    • ID – Patient’s ID
    • SSN – Social security number
    • DRIVERS – Driving License
    • PASSPORT – Passport number
Date fields
    • BIRTHDATE
    • DEATHDATE
Relation fields
    • patients.ID allergies.PATIENT
    • patients.ID  encounters.PATIENT
    • patients.ID  devices.PATIENT
    • patients.ID  conditions.PATIENT
    • patients.ID  medications.PATIENT
    • patients.ID  imaging_studies.PATIENT
    • patients.ID  immunizations.PATIENT
    • patients.ID  procedures.PATIENT
    • patients.ID observations. PATIENT
    • patients.ID  payer_transitions.PATIENT
    • patients.ID  carplans.PATIENT

Table descriptions : Encounters ( Table Name)
This table holds the details of the patient’s diagnosis (SNOMED Code) with its description along with the encounter class (admission class) of each patient.

Columns by description
ID: This column refers to the primary key of this table with each SNOMED code with the patient id created.
START: This column refers to the start date of the encounter class code.
STOP: This column refers to the stop date of the encounter class code.
PATIENT: This column refers to the ID value of the patient’s table each ID.
ORGANIZATION: This column refers to the patient's linked / admitted / appeared location of the hospital.
PROVIDER: This column refers to the physician who addressed the patient’s diagnosis at the visit and it is referred to providers table’s master.
PAYER: This column refers to the patient’s payer details, and it is referred to payers table.
ENCOUNTERCLASS: This column refers to patient’s admission class like inpatient, outpatient, emergency etc.
CODE: This code is used to represent a patient encounter where the primary reason for the visit is the presence of a symptom.
“SNOMED Code - This is a globally recognized clinical terminology used in healthcare to standardize the way patient information is recorded and shared”
DESCRIPTION: This refers to the detailed explanation of the SNOMED code.
BASE_ENCOUNTER_COST: This column refers to the base cost amount to pay either from the payer or patient for the entered encounter class.
TOTAL_CLAIM_COST: This column refers to the total claim cost for the patient’s visit based on the encounter class.
PAYER_COVERAGE: This column refers to the cost of the amount that the payer can cover from the total amount of claim cost.
REASONCODE: This column refers to the actual reason encounter found by the provider at the time of visit.
REASONDESCRIPTION: This column refers to the detailed description of the reason code. Note: This also as a SNMOED code.

Identification fields
    • ID – Encounter’s ID
    • PATIENT – Patient ID
    • CODE – SNMOED code at the visit appearance.
    • REASONCODE – SNOMED code found by physician on diagnosis.
Date fields
    • START
    • STOP
Relation fields
    • encounters.ID  allergies.ENCOUNTER
    • encounters.ID  conditions.ENCOUNTER
    • encounters.ID  devices.ENCOUNTER
    • encounters.ID imaging_studies.ENCOUNTER
    • encounters.ID immunizations.ENCOUNTER
    • encounters.ID medications.ENCOUNTER
    • encounters.ID  careplans.ENCOUNTER
    • encounters.ID  observations.ENCOUNTER
    • encounters.ID  procedures.ENCOUNTER

Table descriptions : Imagining_Studies ( Table Name )

MODALITY_CODE: This column refers, scan options like Xray, CT, Ultrasound in code form.
MODALITY_DESCRIPTION: This column shows detailed information of the prior column.
SOP_CODE: This column refers to SOP Class UID (Service Object Pair Class Unique Identifier) as a unique number that identifies a specific type of DICOM object, such as a CT image or a structured report. It's like a template that defines the structure and rules for a particular type of medical information.
SOP_DESCRIPTION: This column shows detailed information of the prior column.
